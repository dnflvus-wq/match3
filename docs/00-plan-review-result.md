## 공통 문제 5가지

### 1. MCP 도구 호출이 추상적
- `script-execute`라고만 적고, **실행할 C# 코드가 없음**
- `gameobject-component-modify`라고만 적고, **어떤 필드를 어떤 값으로 바꾸는지 없음**
- **필요한 것**: 각 Step에서 실제 실행할 코드 블록 또는 구체적 파라미터 JSON

### 2. 코드가 메서드 시그니처만
- 클래스명과 메서드명만 나열, **구현 로직(루프, 조건문, 알고리즘)이 없음**
- `{ ... }`로 생략된 메서드 다수
- **필요한 것**: 핵심 알고리즘의 의사코드 또는 실제 코드

### 3. 구체적 데이터 값 부족
- "이동수 30~40" 같은 범위만 있고, **레벨별 정확한 테이블 없음**
- SO 에셋의 필드값이 "예시"로만 적혀 있음
- **필요한 것**: Level01~Level05 각각의 정확한 파라미터 테이블

### 4. 테스트 절차/기대값 없음
- 검증 항목에 "XX 정상" 이라고만 적혀 있음
- **어떻게 테스트하고, 어떤 값이면 통과인지** 없음
- **필요한 것**: 테스트 시나리오 + 기대 결과

### 5. 파일 경로/에셋 경로 부족
- "Prefab 할당"이라고만 하고, **Assets/Prefabs/XX/YY.prefab** 경로 없음
- **필요한 것**: 모든 에셋 참조에 정확한 경로

## 권장 수정 방향

### 방법 A: 계획서 완전 재작성 (시간 매우 많이 소요)
- 각 Phase를 "실행 가능한 스크립트" 수준으로 재작성
- 모든 MCP 호출에 실제 파라미터 포함
- 모든 코드에 구현 로직 포함

### 방법 B: 개발하면서 구체화 (현실적 권장)
- 현재 계획서를 "로드맵"으로 활용
- 각 Phase 개발 시작 시 해당 계획서를 상세화
- 실제 MCP 호출 결과를 계획서에 기록
- 다음 Phase 전에 이전 Phase 계획서가 "실행 로그" 수준으로 구체화됨
