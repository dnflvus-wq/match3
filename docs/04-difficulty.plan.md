# Phase 4: 난이도 곡선 설계

## 목표
사인파(Sine Wave) 기반 난이도 곡선으로 유저 리텐션 극대화

## 핵심 개념
- **사인파 곡선**: Hard → Easy → Hard → Easy 반복
- **감정 사이클**: 좌절 → 성취감 → 긴장 → 쾌감
- **동적 난이도 조절(DDA)**: 연속 실패 시 유리한 타일 스폰 확률 증가

## 구체적 설계
- 레벨 1~10: 쉬움 (튜토리얼 구간)
- 레벨 11~20: 점진적 어려움 증가
- 레벨 21+: 사인파 패턴 (3~4레벨 쉬움 → 1~2레벨 어려움 반복)
- Hard 레벨: 이동수 적음, 장애물 많음 → 부스터 소모 유도
- Easy 레벨: 이동수 넉넉, 연쇄 반응 많음 → 쾌감 + 보상

## 구현 방식
- **LevelConfig SO**: 난이도 태그 (Easy/Medium/Hard)
- **가중치 테이블**: 타일 스폰 확률을 난이도별 조정
- **DDA 시스템**: 연속 실패 카운터 → 스폰 확률 보정
- **AI 시뮬레이션** (향후): 헤드리스 모드로 클리어율 자동 측정

## 검증 항목
- [ ] 레벨별 난이도 태그 설정
- [ ] Hard 레벨에서 적절한 도전감
- [ ] Easy 레벨에서 시원한 연쇄 반응
- [ ] 연속 실패 시 DDA 작동
- [ ] 클리어율 30~70% 범위 유지
